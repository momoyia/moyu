# 取消购买状态修复说明

## 问题描述
当用户在购买过程中点击"取消"时，`_isTransactionInProgress` 状态没有正确重置，导致：
1. 所有购买按钮变成加载状态
2. 无法再次发起购买
3. 需要重启应用才能恢复

## 修复内容

### 1. 处理取消状态 (LinstPurchaseManager.dart)

#### 添加取消状态处理
```dart
else if (purchaseDetails.status == PurchaseStatus.canceled) {
  // 用户取消购买
  print('Transaction canceled by user');
  onPurchaseError?.call('购买已取消');
}
```

#### 修复状态重置逻辑
- 将状态重置移到 `else` 块内部
- 确保只有在非 pending 状态时才重置
- 在错误处理中也重置状态

### 2. 优化错误提示 (coin_store_screen.dart)

#### 静默处理取消操作
```dart
void _handlePurchaseError(String error) {
  // 如果是用户取消购买，不显示错误提示
  if (error.contains('取消') || error.toLowerCase().contains('cancel')) {
    return;
  }
  _showMessage('购买失败: $error');
}
```

用户主动取消购买时，不会显示错误提示，提供更好的用户体验。

### 3. 添加超时保护 (LinstPurchaseManager.dart)

#### 30秒超时自动重置
```dart
// 添加超时保护，30秒后自动重置状态
Future.delayed(const Duration(seconds: 30), () {
  if (_isTransactionInProgress) {
    print('Transaction timeout, resetting state');
    _isTransactionInProgress = false;
    _isTransactionPending = false;
  }
});
```

防止因网络问题或其他异常导致状态永久卡住。

### 4. 改进用户交互 (coin_store_screen.dart)

#### 购买冲突对话框
当检测到有购买正在进行时，显示对话框让用户选择：
- **继续等待**：等待当前购买完成
- **取消**：放弃本次购买

```dart
if (_purchaseManager.isTransactionInProgress) {
  final shouldCancel = await showDialog<bool>(
    context: context,
    builder: (context) => AlertDialog(
      title: const Text('购买进行中'),
      content: const Text('当前有一个购买正在进行中，请稍候...'),
      actions: [
        TextButton(
          onPressed: () => Navigator.pop(context, false),
          child: const Text('继续等待'),
        ),
        TextButton(
          onPressed: () => Navigator.pop(context, true),
          child: const Text('取消'),
        ),
      ],
    ),
  );
  
  if (shouldCancel != true) {
    return;
  }
}
```

## 修复的场景

### 场景 1：用户取消购买
**之前**：
1. 用户点击购买
2. iOS 弹出购买确认
3. 用户点击"取消"
4. ❌ 按钮保持加载状态
5. ❌ 无法再次购买

**现在**：
1. 用户点击购买
2. iOS 弹出购买确认
3. 用户点击"取消"
4. ✅ 按钮恢复正常
5. ✅ 可以立即再次购买
6. ✅ 不显示错误提示

### 场景 2：网络超时
**之前**：
1. 用户点击购买
2. 网络请求超时
3. ❌ 状态永久卡住
4. ❌ 需要重启应用

**现在**：
1. 用户点击购买
2. 网络请求超时
3. ✅ 30秒后自动重置
4. ✅ 可以继续使用

### 场景 3：快速连续点击
**之前**：
1. 用户快速点击两次购买
2. ❌ 可能触发两次购买
3. ❌ 或者状态混乱

**现在**：
1. 用户快速点击两次购买
2. ✅ 第一次正常处理
3. ✅ 第二次显示"购买进行中"对话框
4. ✅ 用户可以选择等待或取消

## 状态流转图

```
[空闲] 
  ↓ 点击购买
[进行中] (_isTransactionInProgress = true)
  ↓
  ├─ 购买成功 → [空闲] + 增加金币
  ├─ 购买失败 → [空闲] + 显示错误
  ├─ 用户取消 → [空闲] (不显示提示)
  └─ 超时(30s) → [空闲] (自动重置)
```

## 注意事项

1. **不要删除超时保护**：虽然 30 秒看起来很长，但在网络不好的情况下可能需要这么久
2. **保持状态一致性**：确保所有错误路径都正确重置状态
3. **用户体验优先**：取消购买是正常操作，不应该显示错误提示
4. **防御性编程**：即使理论上不会发生，也要添加超时保护
