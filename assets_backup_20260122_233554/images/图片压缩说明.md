# 图片压缩使用说明

## 🎯 目标
将 assets 文件夹中的图片进行压缩，在保证视觉质量的前提下减小文件体积，提升应用性能。

## 📁 需要压缩的图片文件

### assets/images/ 文件夹
- `1.jpg` - 自然主题背景图片
- `2.jpg` - 户外主题背景图片  
- `3.jpg` - 文化主题背景图片
- `4.jpg` - 娱乐主题背景图片

### assets/natures/ 文件夹
- `nature1.jpg` 到 `nature10.jpg` - 精选专题配图

## 🚀 快速开始

### 方法1: 使用快速脚本 (推荐)
```bash
./compress.sh
```

### 方法2: 直接使用Python脚本
```bash
python3 scripts/compress_images.py
```

### 方法3: 使用Shell脚本 (需要ImageMagick)
```bash
./scripts/compress_images.sh
```

## ⚙️ 压缩配置

当前配置针对移动应用优化：
- **JPEG质量**: 85 (高质量与文件大小的最佳平衡)
- **最大尺寸**: 1200x1200 (适合高分辨率屏幕)
- **PNG优化**: 开启
- **元数据清理**: 开启 (移除EXIF等数据)

## 📊 预期效果

### 文件大小优化
- 原始图片 (2-5MB) → 压缩后 (200KB-1MB)
- 平均减少 60-80% 的文件大小
- 应用包体积显著减小

### 性能提升
- 更快的图片加载速度
- 减少内存占用
- 更好的用户体验

## 🔒 安全保障

- **自动备份**: 压缩前自动创建备份
- **智能压缩**: 只有在文件变小时才替换
- **质量保证**: 使用高质量压缩算法

## 📱 移动端优化

### Flutter应用特点
- 支持多种屏幕密度
- 自动选择合适的图片资源
- 减少首次加载时间

### 用户体验
- 更快的启动速度
- 流畅的图片显示
- 节省用户流量

## 🎨 图片建议

### 推荐的图片规格
- **格式**: JPEG (照片) / PNG (图标)
- **尺寸**: 800x600 到 1200x900
- **质量**: 原始高质量图片 (压缩脚本会自动优化)

### 图片内容建议
1. **自然 (1.jpg)**: 山川、森林、湖泊等自然风光
2. **户外 (2.jpg)**: 登山、徒步、露营等户外活动
3. **文化 (3.jpg)**: 古建筑、博物馆、文化遗址等
4. **娱乐 (4.jpg)**: 音乐会、节庆、娱乐场所等
5. **精选配图**: 各种旅行相关的精美图片
